# Portable SAM Fusion 误检测问题分析报告

**生成时间**: 2026-02-09
**项目**: portable_sam_fusion
**问题描述**: 训练后的模型在推理时产生大量误检测框

---

## 📊 问题概览

### 现象
- 推理结果中存在大量误检测框（False Positives）
- 与 migrated_project 相比，检测质量明显下降
- 可视化结果显示红色预测框数量远超绿色GT框

### 影响范围
- **影响模块**: RPN (Region Proposal Network)
- **影响数据**: 所有测试集
- **严重程度**: 🔴 严重 - 严重影响模型可用性

---

## 🔍 问题诊断

### 问题1: RPN训练严重不足 ⚠️ **最严重的问题**

#### 现象
从训练日志 `logs/rsprompter_fusion_latest_satS_v1.log` 可以看到：

```
Epoch 50-101: loss_rpn_cls ≈ 0.685
```

#### 分析
- 二分类交叉熵的**随机猜测值是 -ln(0.5) = 0.693**
- loss_rpn_cls=0.685 **接近随机水平**
- 说明 RPN 几乎没有学会区分前景和背景

#### 影响
- RPN 负责生成候选框（proposals）
- RPN 训练不好 → 大量误检测框
- 这是导致误检测的**根本原因**

#### 对比
- **migrated_project**: loss_rpn_cls 应远低于 0.685（正常值 0.1-0.3）
- **portable_sam_fusion**: loss_rpn_cls ≈ 0.685（接近随机）

---

### 问题2: 非对称学习率策略失败 🔴 **关键问题**

#### 配置
从 `scripts/start_rsprompter_fusion_v10_satS_v1.sh` 可以看到：

```bash
# 阶段1学习率配置
SAT_BACKBONE_LR_MULT_STAGE1=0.01   # ❌ 太低！
SAT_OTHER_LR_MULT_STAGE1=1.0
DRONE_LR_MULT_STAGE1=1.0

# 如果 base_lr = 2e-4
# sat_backbone 学习率 = 2e-4 × 0.01 = 2e-6
# drone 学习率 = 2e-4 × 1.0 = 2e-4 (相差 100倍！)
```

#### 验证
从训练日志验证：

```
Epoch 100: sat_backbone=2.69e-06, drone_guidance=4.28e-05
```

#### 分析
- **sat_backbone (SAM Vision Encoder) 学习率极低**，几乎没被训练
- **drone_guidance 模块学习率较高**，主导了训练过程
- 导致卫星分支的特征提取能力严重退化
- RPN 依赖卫星特征提取，因此训练不足

#### 根因
设计意图是"稳定融合"，但**过度抑制了卫星分支的学习**
- sat_backbone 学习率太低（0.01倍）
- drone 分支学习率过高（1.0倍）
- 训练失衡，drone 分支主导

---

### 问题3: 语义引导模块在推理时引入噪声 🟡

#### 代码
从 `models/rsprompter_anchor_drone_guidance.py` 可以看到：

```python
def extract_feat(self, batch_inputs):
    # 无人机分支
    drone_bev = self.drone_encoder(...)

    # 卫星分支
    x, image_embeddings, image_positional_embeddings = super().extract_feat(sat)

    # 语义引导融合
    guided_feats, guided_emb = self.guidance(
        feats=x,
        drone_bev=drone_bev,  # ❌ 可能为空或质量差
        image_embeddings=image_embeddings if self.gate_image_embeddings else None,
    )
```

#### 语义引导机制
```python
class DroneSemanticGuidance(nn.Module):
    def forward(self, feats, drone_bev, image_embeddings=None):
        context = self.pool(drone_bev).flatten(1)  # 无人机BEV特征

        guided: List[torch.Tensor] = []
        for x, gate_mlp in zip(feats, self.level_gates):
            x_context = self.pool(x).flatten(1)  # 卫星特征
            combined_context = torch.cat([context, x_context], dim=1)

            gate = gate_mlp(combined_context).unsqueeze(-1).unsqueeze(-1)
            guided.append(x * gate)  # ⚠️ 门控卫星特征
```

#### 分析
- **推理时如果有无人机数据**，drone_bev 可能与卫星场景不匹配
  - 时相差异（拍摄时间不同）
  - 视角差异（俯拍 vs 倾斜）
  - 光照/天气差异

- **语义引导的门控机制**可能：
  - 错误地抑制正确的卫星特征
  - 引入噪声，干扰 RPN 的学习
  - 导致卫星特征退化，RPN 无法正常工作

#### 验证需要
- 检查 drone_bev 的质量
- 对比有/无语义引导的效果
- 验证门控权重是否合理

### 问题4: score_thr=0.05 过低 🟡

#### 配置
从 `configs/rsprompter_anchor_satS_drone_guidance_v1.py` 可以看到：

```python
test_cfg=dict(
    rcnn=dict(score_thr=0.05, ...)  # ❌ 太低！
)
```

#### 分析
- 0.05 的置信度阈值**过于宽松**
- 结合 RPN 训练不好，会产生大量低置信度的误检测
- 即使经过 NMS（IoU=0.5），仍会保留很多误检框

#### 对比
- **migrated_project**: score_thr=0.05（相同）
- **portable_sam_fusion**: score_thr=0.05（相同）

**但是**：
- migrated_project 的 RPN 训练良好，高置信度预测多
- portable_sam_fusion 的 RPN 训练差，低置信度预测多
- 因此需要**提高阈值**来过滤误检测

#### 推理脚本
从 `inference/inference_rsprompter_fusion.py` 可以看到：

```python
parser.add_argument('--show-score-thr', type=float, default=0.3, help='score threshold for visualization')
```

- 可视化默认使用 0.3
- 但模型使用 0.05
- 导致可视化结果包含大量低置信度预测

### 问题5: SAM 模型和特征层选择差异 🟡

#### 对比

| 项目 | SAM 模型 | select_layers | 特征层数量 | 配置文件 |
|------|-----------|---------------|-------------|---------|
| migrated_project | SAM-ViT-Base | range(1, 13, 2) | 6层 | sam-vit-base |
| portable_sam_fusion | SAM-ViT-Base | range(1, 13, 2) | 6层 | sam-vit-base |

#### 实际发现
- 两个项目都使用 **SAM-ViT-Base**（不是配置文件中提到的 Huge）
- 特征层选择**一致**
- SAM 配置文件路径：
  - migrated_project: `./checkpoints/facebook/sam-vit-base`
  - portable_sam_fusion: `./checkpoints/facebook/sam-vit-base`

#### 结论
**不是主要问题**，但需要注意：
- 两个项目使用相同的 SAM 预训练权重（`sam_vit_b_01ec64.pth`）
- 特征提取能力应该相同
- 问题不在于 SAM 模型本身，而在于训练策略

## 🎯 问题根因总结

### 优先级排序

| 问题 | 严重程度 | 影响 | 根因 |
|------|---------|------|------|
| 1. RPN 训练不足 | 🔴 严重 | 产生大量误检框 | sat_backbone 学习率过低 |
| 2. 非对称学习率失败 | 🔴 严重 | 卫星分支退化 | 学习率策略设计不合理 |
| 3. 语义引导引入噪声 | 🟡 中等 | 抑制正确特征 | 无人机与卫星不匹配 |
| 4. score_thr 过低 | 🟡 中等 | 低置信度误检 | 未针对模型优化 |
| 5. SAM 模型差异 | 🟢 轻微 | 非主要问题 | 配置一致 |

### 因果关系链

```
非对称学习率失败
    ↓
sat_backbone 学习率过低 (2e-6)
    ↓
卫星特征提取能力退化
    ↓
RPN 训练不足 (loss_rpn_cls≈0.685)
    ↓
产生大量误检测框
    ↓
语义引导可能加剧问题
    ↓
score_thr=0.05 过低，未能过滤误检测
```

## 💡 解决方案

### 方案 1: 调整学习率策略 ⭐ **最推荐**

#### 目标
让 sat_backbone 充分训练，恢复 RPN 的特征提取能力

#### 修改
```bash
# 文件: scripts/start_rsprompter_fusion_v10_satS_v1.sh

# 阶段1: 平衡学习率
SAT_BACKBONE_LR_MULT_STAGE1="${SAT_BACKBONE_LR_MULT_STAGE1:-0.1}"   # 从 0.01 提高到 0.1 (10倍)
SAT_OTHER_LR_MULT_STAGE1="${SAT_OTHER_LR_MULT_STAGE1:-1.0}"
DRONE_LR_MULT_STAGE1="${DRONE_LR_MULT_STAGE1:-0.5}"                 # 从 1.0 降低到 0.5

# 如果 base_lr = 2e-4
# sat_backbone 学习率 = 2e-4 × 0.1 = 2e-5 (提高 10倍)
# drone 学习率 = 2e-4 × 0.5 = 1e-4 (降低 2倍)

# 阶段2: 进一步提高 sat_backbone
SAT_BACKBONE_LR_MULT_STAGE2="${SAT_BACKBONE_LR_MULT_STAGE2:-0.2}"   # 从 0.05 提高到 0.2 (4倍)
SAT_OTHER_LR_MULT_STAGE2="${SAT_OTHER_LR_MULT_STAGE2:-0.5}"        # 从 0.2 提高到 0.5
DRONE_LR_MULT_STAGE2="${DRONE_LR_MULT_STAGE2:-0.5}"                 # 从 1.0 降低到 0.5
```

#### 预期效果
- sat_backbone 充分训练
- RPN loss_rpn_cls 降至 0.3 以下
- 误检测大幅减少

#### 实施难度
⭐⭐ 简单 - 只需修改 shell 脚本

## 📝 总结

### 核心问题
**非对称学习率策略导致 sat_backbone 学习率过低（2e-6），卫星特征提取能力退化，RPN 训练不足（loss_rpn_cls≈0.685），产生大量误检测。**

### 关键发现
1. RPN loss ≈ 0.685（接近随机猜测 0.693）
2. sat_backbone 学习率 = 2e-6（drone 的 1/100）
3. 语义引导可能引入噪声
4. score_thr=0.05 过低，未能过滤误检测

### 解决路径
1. **立即执行**: 提高 score_thr 到 0.3（5 分钟见效）
2. **短期调试**: 禁用语义引导，对比效果（1 天）
3. **中期优化**: 调整学习率策略 + 优化数据增强，重新训练（1-2 周）

### 预期结果
- RPN loss 降至 0.3 以下
- 误检测大幅减少
- 推理质量显著提升

### 注意事项
- portable_sam_fusion 是独立项目，不能依赖其他项目的权重
- 所有解决方案都基于项目自身的架构和配置
- 重点关注学习率策略和 RPN 训练质量
