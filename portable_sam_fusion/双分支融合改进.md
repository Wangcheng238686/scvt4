# 场景特定对齐与空间融合改进

## 概述

本文档记录了对无人机分支的两个重要改进：
1. **场景特定对齐矩阵**：解决多场景共用一个对齐矩阵的问题
2. **特征级对比空间融合**：解决 BEV 与卫星特征融合时空间信息丢失和时空不一致问题

---

## 一、场景特定对齐矩阵

### 问题
- 所有场景共用一个 `alignment_weight` 矩阵
- 每个场景的 COLMAP 世界坐标系是独立定义的
- 单一矩阵无法适配所有场景的坐标系差异

### 解决方案
使用场景嵌入预测每个场景的对齐矩阵：
- `SceneAwareAlignment` 类：场景嵌入表 + 对齐矩阵预测头
- 每个场景有独立的对齐参数
- 支持未见场景的全局默认对齐

### 修改的文件
| 文件 | 修改内容 |
|------|----------|
| `uav/cross_view_attention.py` | 新增 `SceneAwareAlignment` 类 |
| `uav/factory.py` | 支持 `max_scenes` 参数 |
| `data/satellite_drone_dataset.py` | 添加 `scene_index` 返回 |
| `data/loader.py` | 传递场景映射 |
| `train/train_rsprompter_fusion.py` | 添加 `--max-scenes` 参数 |

---

## 二、特征级对比空间融合

### 问题
- 全局池化丢失 BEV 空间信息
- 无法应对时空不一致

### 解决方案
1. BEV 投影 + 上采样到卫星特征尺寸
2. 空间注意力融合
3. 相似度图识别不一致区域
4. 相似度加权自适应融合
5. 残差细化

### 应对时空不一致
| 机制 | 说明 |
|------|------|
| 相似度图 | 自动识别不匹配区域 |
| 自适应融合 | 高相似度多融合，低相似度保留原特征 |
| 对齐损失 | 训练时增强特征对齐 |

---

## 三、使用方法

### 配置
```python
model = dict(
    type="RSPrompterAnchorDroneGuidance",
    max_scenes=1000,
    use_spatial_fusion=True,
    spatial_fusion_cfg=dict(
        num_heads=4,
        temperature=0.1,
        align_loss_weight=0.1,
    ),
)
```

### 训练
```bash
python train/train_rsprompter_fusion.py \
    --config configs/rsprompter_anchor_satS_drone_guidance_v1.py \
    --use-drone --max-scenes 1000
```
