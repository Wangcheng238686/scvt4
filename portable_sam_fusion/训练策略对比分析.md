## 🔍 关键差异详细分析

### 差异1: 优化器和学习率策略 ⚠️ **最关键差异**

#### migrated_project

**配置**:
```python
# configs/rsprompter_university.py
optim_wrapper = dict(
    type='AmpOptimWrapper',
    dtype='float16',
    optimizer=dict(
        type='AdamW',
        lr=base_lr,  # base_lr = 0.0001
        weight_decay=0.05
    )
)
```

**特点**:
- ✅ **统一学习率**: 所有参数使用相同的学习率
- ✅ **平衡训练**: sat_backbone、RPN、ROI Head 平等学习
- ✅ **简单有效**: 避免复杂的参数组管理

---

#### portable_sam_fusion

**配置**:
```python
# train/train_rsprompter_fusion.py
def _build_optimizer(model, lr, sat_backbone_lr_mult, sat_other_lr_mult, drone_lr_mult):
    params_sat_backbone = []
    params_sat_other = []
    params_drone = []

    for name, param in model.named_parameters():
        if not param.requires_grad:
            continue
        if name.startswith("drone_encoder.") or name.startswith("guidance."):
            params_drone.append(param)
        elif name.startswith("backbone.") or name.startswith("shared_image_embedding."):
            params_sat_backbone.append(param)
        else:
            params_sat_other.append(param)

    param_groups = []
    if params_sat_backbone:
        param_groups.append({
            "params": params_sat_backbone,
            "lr": lr * sat_backbone_lr_mult,  # ❌ 极低
            "name": "sat_backbone"
        })
    if params_sat_other:
        param_groups.append({
            "params": params_sat_other,
            "lr": lr * sat_other_lr_mult,
            "name": "sat_other"
        })
    if params_drone:
        param_groups.append({
            "params": params_drone,
            "lr": lr * drone_lr_mult,  # ❌ 过高
            "name": "drone_guidance"
        })

    return optim.AdamW(param_groups, lr=lr, weight_decay=0.05, eps=1e-6)
```

**实际学习率** (从训练脚本 `start_rsprompter_fusion_v10_satS_v1.sh`):
```bash
# 阶段1配置
base_lr = 2e-4
sat_backbone_lr_mult = 0.01    # ❌ 太低！
sat_other_lr_mult = 1.0
drone_lr_mult = 1.0

# 实际学习率
sat_backbone: 2e-4 × 0.01 = 2e-6  (0.000002)
drone: 2e-4 × 1.0 = 2e-4          (0.0002)
相差: 100倍！
```

**问题分析**:
- ❌ **sat_backbone学习率极低** (2e-6)，几乎未训练
- ❌ **drone学习率过高** (2e-4)，主导训练过程
- ❌ **训练严重失衡**: drone分支学到特征，satellite分支退化
- ❌ **RPN依赖satellite特征**: RPN训练不足（loss_rpn_cls≈0.685）

**训练日志验证**:
```
Epoch 100: sat_backbone=2.69e-06, drone_guidance=4.28e-05
```

### 差异2: 学习率调度策略 ❌ **缺少预热**

#### migrated_project

**配置**:
```python
param_scheduler = [
    dict(
        type='LinearLR',
        start_factor=0.001,  # ✅ 预热因子
        by_epoch=False,
        begin=0,
        end=50  # 前50个iter预热
    ),
    dict(
        type='CosineAnnealingLR',
        eta_min=base_lr * 0.001,  # ✅ 最小学习率 = 0.0000001
        begin=1,
        end=max_epochs,
        T_max=max_epochs,
        by_epoch=True
    )
]
```

**特点**:
- ✅ **Linear预热**: 前50个iter线性增加学习率
  - 从 0.0001 × 0.001 = 0.0000001
  - 到 0.0001 × 1.0 = 0.0001
- ✅ **Cosine退火**: 平滑降低学习率到最小值
- ✅ **稳定训练**: 避免初始阶段的剧烈波动

**学习率曲线**:
```
LR
  ↑
0.0001 |────────────────────
       |       \           \
       |        \           \
0.00005 |        \           \────────────
       |         \                      \
       |          \                      \
0.0000001 |──────────────────────────────\──→ eta_min
       +-------------------------------------→ Iterations
       0      50       100     150    ...
       |预热|           Cosine退火
```

---

#### portable_sam_fusion

**配置**:
```python
# train/train_rsprompter_fusion.py
scheduler = optim.lr_scheduler.CosineAnnealingLR(
    optimizer,
    T_max=args.epochs,  # T_max = 180
    eta_min=args.lr * 0.01  # eta_min = 2e-4 × 0.01 = 2e-6
)
```

**特点**:
- ❌ **无预热阶段**: 初始即使用完整学习率
- ❌ **直接从高LR开始**: 可能导致初始训练不稳定
- ❌ **eta_min过高**: 最小学习率 = 2e-6（仍然较高）
- ❌ **T_max过长**: 180 epochs的退火周期太长

**学习率曲线**:
```
LR
  ↑
2e-4  |────────────────────────────
       |       \                      \
       |        \                      \
1e-4   |         \                      \────────────
       |          \                              \
       |           \                              \
2e-6   |──────────────────────────────────────────────\──→ eta_min
       +---------------------------------------------→ Epochs
       0      45      90      135      180
       |Cosine退火 (无预热)
```

**问题分析**:
- ❌ 初始训练可能不稳定（无预热）
- ❌ sat_backbone学习率本就极低（2e-6），退火到eta_min后几乎不变
- ❌ 学习率下降缓慢，训练效率低

from torch.optim.lr_scheduler import SequentialLR, LinearLR, CosineAnnealingLR

# 预热调度器

warmup_scheduler = LinearLR(
    optimizer,
    start_factor=0.001,
    total_iters=warmup_epochs

    optimizer,
    T_max=args.epochs - warmup_epochs,

)

scheduler = SequentialLR(
    optimizer,
    schedulers=[warmup_scheduler, main_scheduler],

)

## 🎯 结论

### 核心问题

**portable_sam_fusion的训练策略存在致命缺陷：非对称学习率策略导致sat_backbone学习率极低（2e-6），卫星特征提取能力退化，RPN训练不足（loss_rpn_cls≈0.685），最终导致大量误检测。**

### 次要问题

1. **数据增强严重不足**: 几乎无增强，泛化能力弱
2. **缺少学习率预热**: 初始训练不稳定，收敛慢
3. **无混合精度训练**: 训练效率低
4. **训练轮数过多**: 浪费计算资源

|------|------------------|---------------------|------|

| 学习率调度 | ✅ 预热+退火 | ❌ 仅退火 | portable差 |
| 混合精度 | ✅ AMP | ❌ 无 | portable差 |
| 梯度裁剪 | ❌ 无 | ✅ 有 | portable优 |
| 早停机制 | ❌ 无 | ✅ 有 | portable优 |
