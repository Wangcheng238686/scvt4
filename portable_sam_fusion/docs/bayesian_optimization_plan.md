# 贝叶斯优化调参方案

## 一、调参参数

| 参数 | 搜索范围 | 参数类型 |
|------|----------|----------|
| `LEARNING_RATE` | [1e-5, 5e-4] | log均匀分布 |
| `SAT_BACKBONE_LR_MULT` | [0.1, 0.8] | 均匀分布 |
| `DRONE_LR_MULT` | [0.2, 1.0] | 均匀分布 |
| `align_loss_weight` | [0.05, 0.3] | 均匀分布 |

### 参数说明

- **LEARNING_RATE**: 基础学习率，影响模型收敛速度和最终性能
- **SAT_BACKBONE_LR_MULT**: 卫星backbone学习率倍率，预训练backbone需要较小学习率避免破坏预训练权重
- **DRONE_LR_MULT**: 无人机分支学习率倍率，控制无人机语义引导分支的训练强度
- **align_loss_weight**: 对齐损失权重，控制卫星-无人机特征对齐的强度

## 二、优化设置

| 配置项 | 推荐值 |
|--------|--------|
| **代理模型** | 高斯过程 (GP) |
| **优化框架** | Optuna + GPSampler |
| **优化方向** | minimize (最小化验证loss) |
| **试验次数** | 15-20 次 |
| **每次评估epochs** | 10 epochs |
| **验证集比例** | 10% |

### 代理模型选择理由

高斯过程(GP)适合本项目的原因：
- 低维参数空间（4个参数）
- 小样本场景（15-20次试验）
- 能很好量化不确定性
- 提供置信区间，便于探索-利用平衡

## 三、数据划分策略

```
原始训练集
    │
    ├── 随机划分（固定随机种子，保证可复现）
    │
    ├── 90% → 调参训练集（用于贝叶斯优化阶段的训练）
    │
    └── 10% → 调参验证集（用于贝叶斯优化阶段的评估）
```

### 划分原则

- 使用固定随机种子（如 `seed=42`）确保每次划分一致
- 划分后保存索引，确保可复现
- 验证集在调参阶段**不参与训练**，仅用于评估

---

## 四、两阶段流程

### 核心思想

```
用 90% 的数据来"选方案"，用 100% 的数据来"出成品"
```

### 阶段一：贝叶斯优化搜索

```
数据：90% 训练 + 10% 验证
目标：最小化验证集 Loss
配置：
├── 15-20次试验
├── 每次训练10 epochs
├── 每个epoch结束后在验证集上评估
└── 以最终验证集Loss作为Optuna的objective
```

**优势：**
- 验证集Loss比训练Loss更能反映泛化能力
- 避免选择过拟合的参数组合

### 阶段二：完整训练

```
数据：100% 全量数据（合并调参训练集 + 调参验证集）
目标：获得最终模型
配置：
├── 使用阶段一找到的最优参数
├── 完整训练50 epochs
└── 无验证集（全部数据用于训练）
```

**优势：**
- 不浪费数据，充分利用所有样本
- 最优参数已在调参阶段验证，可直接用于全量训练

## 六、评估策略

### 阶段一：调参阶段

```
目标函数：第10个epoch的验证集Loss
```

**评估指标：**
- **主要指标**: 验证集总Loss
- **记录指标**: 各分项loss（rpn_loss, roi_loss, contrastive_loss等）

### 阶段二：完整训练

```
无验证集，仅监控训练Loss
```

## 十二、方案优势总结

| 对比项 | 原方案 | 改进方案 |
|--------|--------|----------|
| 评估指标 | 训练Loss | 验证集Loss |
| 泛化能力评估 | 无 | 有 |
| 数据利用率 | 调参时浪费10% | 最终训练时100%利用 |
| 过拟合风险 | 较高 | 较低 |
| 参数可靠性 | 一般 | 更可靠 |
